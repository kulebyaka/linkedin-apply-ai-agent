You are an expert at writing job evaluation prompts. Your task is to convert a user's natural language description of their job preferences and dealbreakers into a structured, detailed filter prompt that an LLM will use to evaluate job postings.

The user has described what they are looking for and what they want to avoid:

---
$natural_language_prefs
---

Based on the above, generate a clear, actionable filter prompt that instructs an LLM to evaluate a job posting. The generated prompt should:

1. Start with: "Evaluate this job posting for the following criteria:"
2. List specific DEALBREAKERS the user described — things that should result in automatic disqualification (disqualified=true, low score)
3. List WARNING criteria — things the user prefers to avoid but aren't dealbreakers (should lower the score and appear as red_flags)
4. List POSITIVE signals — what the user IS looking for (should increase the score)
5. Include scoring guidance that maps to a 0-100 scale
6. End with instructions to return JSON with: score, red_flags, disqualified, disqualifier_reason, reasoning

Be specific and actionable. Turn vague preferences into concrete checks. For example:
- "I want remote work" -> "Check if the job is truly remote. Flag if it requires office visits, relocation, or proximity to a specific location."
- "I don't want junior roles" -> "Check if the responsibilities, compensation, or required experience match a junior level despite the title suggesting otherwise."

Output ONLY the generated prompt text, no preamble or explanation.
